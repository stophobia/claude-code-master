## ⭐ 핵심 개념

### 🤖 에이전트

서브에이전트는 제한된 범위의 위임된 작업을 처리합니다:

```markdown
---
name: code-reviewer
description: 품질, 보안, 유지보수성을 위한 코드 리뷰
tools: Read, Grep, Glob, Bash
model: opus
---

당신은 시니어 코드 리뷰어입니다...
```

### 🎯 스킬

스킬은 커맨드나 에이전트가 호출하는 워크플로우 정의입니다:

```markdown
# TDD 워크플로우

1. 먼저 인터페이스 정의
2. 실패하는 테스트 작성 (RED)
3. 최소한의 코드 구현 (GREEN)
4. 리팩토링 (IMPROVE)
5. 80%+ 커버리지 확인
```

### 🪝 훅

훅은 도구 이벤트에서 실행됩니다:

```json
{
  "matcher": "tool == \"Edit\" && tool_input.file_path matches \"\\\\.(ts|tsx|js|jsx)$\"",
  "hooks": [{
    "type": "command",
    "command": "grep -n 'console\\.log' \"$file_path\""
  }]
}
```

### 📏 규칙

규칙은 항상 따라야 하는 가이드라인입니다:

```
~/.claude/rules/
  security.md      # 하드코딩된 시크릿 금지
  coding-style.md  # 불변성, 파일 제한
  testing.md       # TDD, 커버리지 요구사항
```

## 🚀 빠른 시작

```bash
# 저장소 클론
git clone https://github.com/YOUR_USERNAME/claude-code-master.git

# 에이전트를 Claude 설정에 복사
cp claude-code-master/agents/*.md ~/.claude/agents/

# 규칙 복사
cp claude-code-master/rules/*.md ~/.claude/rules/

# 커맨드 복사
cp claude-code-master/commands/*.md ~/.claude/commands/

# 스킬 복사
cp -r claude-code-master/skills/* ~/.claude/skills/
```
