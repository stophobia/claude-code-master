# 기여하기

Claude Code 마스터 프로젝트에 기여해 주셔서 감사합니다!

## 기여 방법

### 1. 이슈 제출

버그를 발견하거나 새로운 기능을 제안하고 싶다면 GitHub Issues를 사용해 주세요.

**버그 리포트 시 포함할 내용:**
- 재현 단계
- 예상 동작
- 실제 동작
- 환경 정보 (OS, Claude Code 버전)

**기능 제안 시 포함할 내용:**
- 사용 사례
- 제안하는 해결 방법
- 대안 고려사항

### 2. Pull Request

<Callout type="info">
  PR을 제출하기 전에 관련 이슈를 먼저 생성해 주세요.
</Callout>

**PR 프로세스:**
1. 저장소 Fork
2. 기능 브랜치 생성 (`feature/amazing-feature`)
3. 변경사항 커밋
4. 브랜치 Push
5. PR 생성

### 3. 문서 개선

문서 개선도 환영합니다:
- 오타 수정
- 예시 추가
- 번역 개선
- 새로운 가이드 작성

## 개발 환경 설정

```bash
# 저장소 클론
git clone https://github.com/revfactory/claude-code-master.git
cd claude-code-master

# 문서 사이트 실행
cd docs
npm install
npm run dev
```

## 커밋 메시지 규칙

커밋 메시지는 **한글**로 작성합니다.

**형식:**
```
<타입>: <설명>

<선택적 본문>
```

**타입:**
| 타입 | 설명 |
|------|------|
| `feat` | 새 기능 |
| `fix` | 버그 수정 |
| `docs` | 문서 수정 |
| `refactor` | 리팩토링 |
| `test` | 테스트 |
| `chore` | 기타 |

**예시:**
```
feat: 새로운 에이전트 추가

- performance-analyzer 에이전트 구현
- 관련 문서 작성
```

## 코드 스타일

### 마크다운

- 헤딩은 순차적으로 사용 (`#` → `##` → `###`)
- 코드 블록에 언어 지정
- 링크는 설명적인 텍스트 사용

### MDX

- Nextra 컴포넌트 활용 (`Callout`, `Tabs`, `Cards`)
- 일관된 들여쓰기
- 적절한 공백

## 디렉토리 구조

```
claude-code-master/
├── agents/           # 에이전트 마크다운
├── commands/         # 커맨드 마크다운
├── rules/           # 규칙 마크다운
├── skills/          # 스킬 마크다운
├── docs/            # Nextra 문서 사이트
│   ├── pages/       # MDX 페이지
│   └── public/      # 정적 파일
└── README.md
```
